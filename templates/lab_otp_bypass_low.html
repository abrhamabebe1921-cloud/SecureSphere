OTP Bypass — Client-Side Logic

Low Auth Bypass · A07:2021 OWASP Top 10
🎯 Objective
What is Client-Side OTP Bypass?
When an application generates an OTP and embeds it directly in the JavaScript/HTML sent to the browser, the security check is done client-side only. An attacker can open DevTools (F12), find the OTP in the page source or network response, then simply enter it — or bypass the check entirely by manipulating JS logic.

Goal: Find the OTP using DevTools and use it to authenticate without having legitimate access to the registered phone or email.
🔍 How To Find The OTP
// Press F12 → Console tab → Type:
otp // returns the OTP value (it's a JS variable!)

// Or view page source: Ctrl+U → Search for "const otp"

// Or Network tab → XHR → look for /api/send-otp response
JSON.parse‍(response).otp
💻 Vulnerable Source Code
<?php
// source/low.php

if (session_status() === PHP_SESSION_NONE) {
    session_start();
}

if (!isset($_SESSION['otp'])) {
    $_SESSION['otp'] = rand(1000, 9999);
}

$otp = $_SESSION['otp'];

$html = "
<form id='otpForm'>
    <input
      id='otpInput'
      style='height:40px; width:200px; padding-left:20px;'
      name='otp'
      required
      placeholder='Enter OTP'
      autocomplete='off'
    >
    <button style='margin-left:10px; width:100px; height:45px;' type='submit'>Verify</button>
</form>

<p id='resultMsg'></p>

<script>
  const otp = '$otp'; // OTP visible here, attacker can see this in dev tools

  document.getElementById('otpForm').addEventListener('submit', function(e) {
    e.preventDefault();
    const userInput = document.getElementById('otpInput').value.trim();

    const resultMsg = document.getElementById('resultMsg');
    if (userInput === otp) {
      resultMsg.style.color = 'green';
      resultMsg.textContent = 'Verified';
      // Optionally clear OTP here or do other logic
    } else {
      resultMsg.style.color = 'red';
      resultMsg.textContent = 'Not Verified';
    }
  });
</script>
";

$page['body'] .= "<div class='body_padded'><h2>Low Level - Client Side OTP</h2>{$html}</div>";
?>

🛡️ How To Defend
Always validate OTPs server-side, never in JavaScript:
• Generate OTPs on the server — never send them in the HTTP response body or JS
• Store OTPs in the server session tied to the user's IP and cookie session ID
• Implement strict rate-limiting: max 3 attempts, then lock the account temporarily
• Set a short TTL: 60–120 seconds maximum validity window
• Use a cryptographically secure RNG: secrets.randbelow(10000) (Python) or openssl_random_pseudo_bytes (PHP)
• Invalidate the OTP immediately after first successful use
