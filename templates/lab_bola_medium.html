Broken Object-Level Authorization (BOLA)

Medium BOLA / API
🎯 Objective
The server now checks your role to authorize access to other users' orders. However, it reads your role from a user-controlled HTTP header: X-User-Role. Since we cannot easily manipulate headers in this simple UI demo, we've provided a field to simulate setting that header for the request. Bypass the check to view Order #102.
📦 Order Viewer — Weak Authorization Check

For this Medium lab, simulating header injection:

To complete this attack in a real environment, you would use Burp Suite to change the X-User-Role header to admin.
