Exploitation Guide
  1. Log in to get an RS256 signed token.
  2. Obtain the server's Public Key (via JWKS endpoint).
  3. Modify token header to use HS256.
  4. Sign the token using the Public Key PEM as the HMAC secret.
  5. If the server uses verify() indiscriminately, it will use the public key as a symmetric secret.
Vulnerable Logic (Confused Deputization)
// The logic below fails by using a generic verify function // that respects the user-provided 'alg' header. function verify($token, $key) { $alg = get_alg($token); if ($alg === 'RS256') { return rsa_verify($token, $key); // $key is treated as Public Key } else if ($alg === 'HS256') { return hmac_verify($token, $key); // $key is treated as Shared Secret! } }
