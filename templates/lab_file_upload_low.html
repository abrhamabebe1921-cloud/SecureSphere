Unrestricted File Upload

Low File Upload · A04:2021 OWASP Top 10
🎯 Objective
What is Unrestricted File Upload?
When an application allows users to upload files without validating the type, extension, or content, an attacker can upload a malicious file (e.g. a PHP web shell). If the server also serves the uploaded file, the attacker can execute arbitrary commands.

Goal: Upload any file type (try .php, .html, or a script) — this Low level applies no restriction whatsoever. Notice that the file is stored and accessible via a direct URL.
⚠️ Attack Scenario
Example Web Shell Upload:
Create a file shell.php containing:

<?php system($_GET['cmd']); ?>

After upload, navigate to the returned URL and append ?cmd=ls or ?cmd=cat /etc/passwd to execute arbitrary OS commands on the server.

🛡️ How To Defend
Secure File Upload Checklist:
• Accept only whitelisted extensions: jpg, jpeg, png, gif, pdf
• Validate MIME type by reading magic bytes (not just $_FILES['type'])
• Rename the file server-side to a random UUID — never trust the original filename
• Store uploads outside the web root and serve through a proxy script
• Scan uploaded files with an antivirus / ClamAV integration
• Set a strict Content-Type header and never execute uploaded files
