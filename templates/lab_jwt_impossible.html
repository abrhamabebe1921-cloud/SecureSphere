🛡️ Secure JWT Implementation
At this level, the server enforces all of the following:
  • Algorithm is hardcoded to RS256 — the alg claim in the token header is completely ignored
  • Tokens are verified with a public key pinned server-side
  • The kid header is validated against a whitelist — no path traversal possible
  • Tokens have a short expiry (15 minutes)
  • Signature is verified before any claims are read
# SECURE implementation:
jwt.decode(token, PUBLIC_KEY, algorithms=["RS256"], options={"require": ["exp", "iat"]})
