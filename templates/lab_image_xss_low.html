XSS via SVG Image Upload

Low XSS · A03:2021 OWASP Top 10
🎯 Objective
What is XSS via Image Upload?
SVG (Scalable Vector Graphics) files are XML-based and can contain embedded <script> tags. A vulnerable application that renders uploaded SVGs directly in the browser — without sanitization — allows attackers to execute arbitrary JavaScript in the victim's session.

Goal: Upload an SVG file containing a <script> tag. When the browser renders the SVG in the iframe, the script will execute — demonstrating stored XSS via file upload.
💡 Craft Your Payload
<svg xmlns="http://www.w3.org/2000/svg">
  <script type="text/javascript">
    alert('XSS via SVG — Flag: EACA{svg_xss_pwned}');
  </script>
</svg>

🛡️ How To Defend
Never render SVG files directly in the browser.
• Re-encode all image uploads as PNG/JPEG server-side (strips embedded scripts)
• Use strict Content-Security-Policy: script-src 'self' to block inline scripts
• Validate MIME type by reading file magic bytes, not just the extension
• Use a Content-Disposition header to force downloads instead of inline rendering
• Sandbox any iframe rendering untrusted content with sandbox=""
